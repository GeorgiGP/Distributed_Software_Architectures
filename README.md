# P2P Архитектури при Разпределено Динамично Балансиране

Курсов проект по **Разпределени Софтуерни Архитектури**

## Описание

Проектът изследва Peer-to-Peer (P2P) архитектури при разпределено динамично балансиране на паралелни приложения. Анализират се две топологии на процесите:

1. **Верижна топология (Chain)** - всеки процес комуникира само със съседите си
2. **Пълен граф (All-to-All)** - всеки процес може да комуникира с всеки друг

## Структура на проекта

```
RSA/
├── docs/
│   └── project.md       # Основен документ на проекта
├── src/
│   ├── static_balancing.c   # Статично циклично балансиране (baseline)
│   ├── p2p_chain.c          # P2P с верижна топология
│   ├── p2p_full.c           # P2P с пълен граф
│   └── Makefile
├── results/                  # Резултати от експериментите
├── run_experiments.sh       # Скрипт за провеждане на тестове
└── README.md
```

## Изисквания

- C компилатор (gcc/clang)
- OpenMPI или MPICH

### Инсталация на macOS

```bash
brew install open-mpi
```

### Инсталация на Ubuntu/Debian

```bash
sudo apt-get install openmpi-bin libopenmpi-dev
```

## Компилация

```bash
cd src
make all
```

## Изпълнение

### Единичен тест

```bash
# Статично балансиране с 4 процеса
mpirun -np 4 ./static_balancing 38

# P2P верижна топология с 4 процеса
mpirun -np 4 ./p2p_chain 38

# P2P пълен граф с 4 процеса
mpirun -np 4 ./p2p_full 38
```

### Пълен тест

```bash
cd ..
./run_experiments.sh
```

## Очаквани резултати

При нехомогенно натоварване (каквото е рекурсивното изчисление на Фибоначи), динамичното P2P балансиране показва значително по-добро ускорение в сравнение със статичното балансиране.
